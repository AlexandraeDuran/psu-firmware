5.10. MMEMory

The MMEMory commands are used to store, read or delete file in the PSU’s SD card. It can also query SD card information. In addition it is used for storing and recalling values used by [SOURce[<n>]]:LIST subsystem. File and directory (folder) names cannot contain the following characters:\ / : * ? " < > |

 

SCPI command

Description

MMEMory

 

:CATalog [<directory>]

Returns a list of items in the specified directory (folder)

:LENgth [<directory>]

Returns the number of items in the specified directory

:CDIRectory {<directory>}

Changes the current directory

:COPY {<source>}, {<destination>}

Copies <source> to <destination>

:DATE? {<filename>}

Returns date that the specified file was last saved

:DELete {<filename>}

Deletes an existing file

Downloads data from the host computer

Creates or opens the specified filename for download data

Loads the instrument setup

Sets write protection

5.10.1. MMEMory:CATalog

Syntax

MMEMory:CATalog? [<directory>]

Description

Returns the SD card usage and availability. Also returns the list of files and directories (folders) names, types and sizes in the current or specified directory. Number of items (files/directories) corresponds to the value returned by the MMEMory:CATalog:LENgth? command.

 

To read out the information in the root directory (folder), specify "\" (backslash) or "/". If <directory> is not set, this function is applied to the current directory. If directory=<path>, this function is applied to <current directory>\<path>.

 

Error occurs if the specified directory does not exist or is set to hidden or system.

Parameters

Name

Type

Range

Default

<directory>

Quoted string

Directory name, either / (slash) or \ (backslash) can be used as the path separator. 1 to 255 characters

–

Return

Returns <UsedMemory>, <FreeMemory>, <FileList> where:

  • UsedMemory – total amount of storage currently used in the directory in bytes,
  • FreeMemory – total amount of storage available in the directory, in bytes) and
  • FileList – all files in the directory as list of comma delimited quoted strings of <filename>, <filetype> and <filesize>.

The following file types are supported:

  • ASC – textual data
  • BIN – binary data
  • FOLD – directory (folder)
  • STAT – instrument (setting) state or user profiles (file extension .conf)
  • TRAC – trace (display) or logged data (file extension .list)

Files stored on the SD card are organized in the following directories tree:

 

\

\

\

Many vendors use “drive letter” for various media like USB: or INT:

Maybe we should use INT: (internal) for SD card.

Usage example

MMEM:CAT? "USER\TEST"

450440512,5000133440,"test.001,BIN,33400","test.002,BIN,1203","old_test1,BIN,440550","\OLD,FOLD,12450","\old2,FOLD,4450036"

 

MMEMCAT? "USER\TEST\test*"

450440512,5000133440,"test.001,BIN,33400","test.002,BIN,1203"

Errors

-250,"Mass storage error"

-251,"Missing mass storage"

-252,"Missing media"

-256,"File name not found"

Related Commands

MMEMory:CATalog:LENgth?

MMEMory:DATE?

MMEMory:TIME?

5.10.4. MMEMory:COPY

Syntax

MMEMory:COPY {<source>}, {<destination>}

Description

Makes a copy of an existing file in the current directory. The file names must include any file extension.

If <destination> is a file name, the copy file is created in the current directory. When <destination> is a <path> (e.g. “test lists\test022”) the source file is duplicated in <current directory>\<path>.

Parameters

Name

Type

Range

Default

<source>

Quoted string

Source file name, 1 to 255 characters

–

<destination>

Quoted string

Copy file name or directory name, either / (slash) or \ (backslash) can be used as the path separator. 1 to 255 characters

–

Usage example

MMEM:COPY "test.bin", "new2\test_new.bin"

Errors

-253,"Corrupt media"

-254,"Media full"

-256,"File name not found"

-258,"Media protected"

Related Commands

MMEMory:CATalog?

MMEMory:CDIRectory

MMEMory:LOCK

5.10.7. MMEMory:DOWNload:DATA

Syntax

MMEMory:DOWNload:DATA {#<length>#<encoding>#<data>}

Description

Downloads data from the host computer to a file in the SD card. The filename must have been previously specified by MMEMory:DOWNload:FNAMe.

The data in <block> is written to the select file, and any data previously stored in the file is lost.

 

While transfer is in progress all channels will be turned off similar to standby mode.

Parameters

Name

Type

Range

Default

<length>

Numeric (NR1)

Define max length!!!

–

<encoding>

Discrete

BASE64, BIN

BASE64

<block>

Discrete

Only exact size is allowed as defined with <length> and <encoding>

–

Usage example

Downloads text Hello world encoded as Base64 and store into the file "test file" in the current directory:

MMEM:DOWN:FNAM "test file"

MMEM:DOWN:DATA #16,BASE64,SGVsbG8gd29ybGQ=

Errors

-253,"Corrupt media"

-254,"Media full"

-258,"Media protected"

We need here an error if communication is interrupted? Maybe we can use for that one of standard SCPI erros for block data: -160,"Block data error" or -161,"Invalid block data"

Related Commands

MMEMory:DOWNload:FNAMe
